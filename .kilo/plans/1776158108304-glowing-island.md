# Plan: Semantic & Document Structure Chunking for DocuRAG

## Goal
Implement two advanced chunking strategies that respect semantic boundaries (sentences, paragraphs) instead of pure character-based splitting.

---

## 1. New Chunker Strategies

### Strategy A: `SemanticChunker`
Splits text into sentences first, then groups sentences into chunks.

**Algorithm:**
1. Split text into sentences using regex: `(?<=[.!?])\s+`
2. Group consecutive sentences until `chunkSize` (chars) would be exceeded
3. If single sentence exceeds `chunkSize`, fall back to RecursiveCharacterTextSplitter logic
4. Overlap: carry last N sentences of chunk to next chunk

**Separators (recursive):**
1. `\n\n` — paragraph boundaries (highest priority)
2. `. ` or `. \n` — sentence endings
3. `; ` — clause separators
4. ` ` — word boundaries (last resort)

### Strategy B: `DocumentStructureChunker`
Respects document hierarchy (headings, paragraphs, lists) for structured documents.

**Algorithm:**
1. Detect structural elements: headings, paragraphs, list items
2. Build hierarchical tree of content
3. Leaf nodes become chunks; internal nodes can be merged up to `chunkSize`
4. Fall back to recursive splitting if node exceeds `chunkSize`

**Separators:**
1. `\n\n` — paragraph breaks
2. `\n` — line breaks (may indicate list items or headings)
3. `\t` — indented content
4. Characters

## 4. Key Algorithms

### SemanticChunker: Sentence Splitting

```java
static final String SENTENCE_SEPARATOR = "(?<=[.!?])\\s+";

static List<String> splitIntoSentences(String text) {
    if (text == null || text.isBlank()) {
        return Collections.emptyList();
    }
    return Arrays.asList(text.split(SENTENCE_SEPARATOR));
}

static List<String> semanticChunks(String text, int chunkSize, int overlap, int minChars) {
    List<String> sentences = splitIntoSentences(text);
    if (sentences.isEmpty()) {
        return Collections.emptyList();
    }
    
    List<String> chunks = new ArrayList<>();
    StringBuilder current = new StringBuilder();
    
    for (String sentence : sentences) {
        // If adding sentence exceeds chunk size
        if (current.length() + sentence.length() + 1 > chunkSize) {
            // Save current chunk if it meets minimum
            if (current.length() >= minChars) {
                chunks.add(current.toString().trim());
            }
            // Start new chunk with overlap (carry last sentence)
            String overlapText = getOverlapText(current.toString(), overlap);
            current = new StringBuilder(overlapText);
            current.append(sentence).append(" ");
        } else {
            current.append(sentence).append(" ");
        }
    }
    
    // Don't forget last chunk
    if (current.length() >= minChars) {
        chunks.add(current.toString().trim());
    }
    
    return chunks;
}

static String getOverlapText(String chunk, int overlapChars) {
    if (overlapChars <= 0 || chunk.isEmpty()) {
        return "";
    }
    int start = Math.max(0, chunk.length() - overlapChars);
    return chunk.substring(start);
}
```

### Handling Oversized Sentences
If a single sentence exceeds `chunkSize`, fall back to recursive character splitting within the sentence:
```java
if (sentence.length() > chunkSize) {
    List<String> subChunks = recursiveCharacterChunks(sentence, chunkSize, overlap, minChars);
    chunks.addAll(subChunks);
}
```

### DocumentStructureChunker: Paragraph Detection

```java
static List<Chunk> documentStructureChunks(String text, int chunkSize, int overlap, int minChars) {
    // Split by double newlines for paragraphs
    String[] paragraphs = text.split("\\n\\n");
    List<Chunk> chunks = new ArrayList<>();
    
    for (String para : paragraphs) {
        if (para.length() <= chunkSize) {
            chunks.add(new Chunk(para, Type.PARAGRAPH));
        } else {
            // Recursively split oversized paragraph
            chunks.addAll(recursiveSplit(para, chunkSize, minChars));
        }
    }
    
    return mergeChunks(chunks, chunkSize, overlap, minChars);
}
```

## 7. Decisions

- **Default Strategy:** `SEMANTIC` — recommended for Q&A use cases, preserves complete sentences
- **Backward Compatibility:** `characterChunks()` will be kept but marked `@Deprecated`
- **Token Estimation:** Continue using `length / 4` (consistent across strategies)
- **Min Chunk Chars:** Applied per-chunk (not per-sentence) in semantic mode
